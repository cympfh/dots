# Check Skill

コードの動作・意図・バグを調査して報告する。**実装・修正・変更は絶対にしない。**

## ルール（絶対守ること）

1. **調査・読み取り・実行のみ許可** — ファイル編集、コード生成、修正提案の実装は禁止
2. **確認結果を正確に報告** — 推測は推測と明記、事実は根拠（ファイルパス・行番号）を示す
3. **修正が必要な場合** — 「修正が必要か？」とユーザーに確認を取って終わる。勝手に直さない

## 報告フォーマット

```
## 確認結果

**何をしているか**
<コードの動作・意図の説明>

**問題の有無**
- バグあり / なし / 要確認

**詳細**
<ファイルパス:行番号 を含む具体的な説明>

**結論**
<1〜2文の要約。修正が必要な場合は「修正しますか？」で締める>
```

## やること / やらないこと

| やること | やらないこと |
|---|---|
| コードを読む | コードを編集する |
| ログ・テストを実行して動作確認 | 修正コードを書く |
| バグの原因を特定して説明 | 「修正しました」と報告する |
| 問題の有無を判断して報告 | ユーザーが頼んでいない変更をする |

## 例

**入力**: `/check src/parser.rs:42 ここでパニックしない？`

**出力**:
```
## 確認結果

**何をしているか**
src/parser.rs:42 で `unwrap()` を呼び出している。入力が None の場合にパニックする実装。

**問題の有無**
- バグあり（条件付き）

**詳細**
src/parser.rs:42: `let val = map.get("key").unwrap();`
`map` に "key" が存在しない場合、`unwrap()` がパニックを起こす。
呼び出し元（src/main.rs:15）ではキーの存在チェックをしていない。

**結論**
入力によってはパニックする。修正しますか？
```
