# Git スキル

毎ターンユーザーから指示があるまでは勝手にこのスキルを使用しないこと

## /git commit

ちょうど一つの commit を作成する

1. git diff を実行し、変更点を確認する
2. 内容を見て git commit を実行する

## /git commits

2つ以上の内容の commit を作成する

1. git diff を実行し 変更点を確認する
2. 内容を見て論理的な粒度に分割する
    - ファイル単位で分割可能な場合にのみ、個別の commit を作成する
    - 分割できない場合は、まとめて一つの commit を作成し、コメントに変更点を詳細に記述する
