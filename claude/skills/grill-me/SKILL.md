# grill-me


プラン・設計・意思決定のあらゆる側面について、共通の理解に達するまでユーザーに徹底的に AskUserQuestion で質問する。甘い回答や曖昧な返答には突っ込む。設計ツリーの各分岐をたどり、判断の依存関係を一つずつ解決する。「幅」より「深さ」を優先する。新たな洞察が得られなくなるまで1つのテーマを掘り下げてから、次へ進む。

質問は必ず一度に一つずつ行うこと。質問には、推奨回答も提示する。
各質問の前に、コードベースの関連箇所を調べる。調べればわかることは聞かない。

## 引数

`/grill-me [n]` の形式で呼び出す。`n`（整数）は質問して良い個数の**目安**。省略時は **5**。

- 目安であり厳密な上限ではない。論点を掘り尽くす前に n 問に達したら、その時点で一旦止まる。
- 止まった時点で「まだ掘るべき論点」を具体的に列挙し、続けてよいか・上限を増やしてよいかをユーザーに確認する。
- ユーザーが続行を認めたら、新たな上限（例: n+3、または無制限）を決めて続ける。認めなければ、その時点までの内容で「まとめ」に入る。
- n 問に達する前に論点を掘り尽くした場合は、そのまま通常の終了提案（下記）に進む。増やす確認は不要。

質問フォーマット:
```
### Q[番号]: [質問文]

[なぜこの質問が重要か]

- **A** — [選択肢]
- **B** — [選択肢]
- ...

**推奨: [A/B/...]** — [理由]
```

論点を掘り尽くしたら終了を提案する。続行を求められれば続ける。

終了時のまとめフォーマット:
```
## まとめ

### 決まったこと
- ...
```
