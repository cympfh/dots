# Todo スキル

Claude Code のための TODO 管理スキルです。`TODO.md` を読み取り、タスクを自律的に実行します。

## 基本的な動作

1. `TODO.md` を読み込む
2. 未完了タスク（`[ ]`）を確認する
3. 優先度・依存関係を考慮して実行順序を決定する
4. タスクを上から順に実行する（ただし優先度や依存関係によって順番を入れ替えることも許される）
5. 実装完了後、`TODO.md` を更新して完了日付時刻を記入する (`Bash(date "+%Y-%m-%d %H:%M")`)

## TODO.md のフォーマット

```markdown
@CLAUDE TODO List

## [x] データベースの設計と実装 [2026-01-01 12:32 完了]
データベースはAWS RDSを使用して、ユーザーデータとタスクデータを管理する。

## [ ] ユーザー認証の実装
JWT を使った認証を実装する。
```

- `[ ]` : 未完了タスク
- `[x]` : 完了タスク（完了日付を `[YYYY-MM-DD HH:MM 完了]` 形式で記入）
- タスクの詳細や変更内容はタスクの説明として記録する

## タスク実行のルール

- **基本は上から順に実行**するが、以下の場合は順番を変えることができる
  - 優先度が明示されている場合
  - 実装の依存関係がある場合（先に完了すべきタスクがある場合）
- タスク完了後は必ず `TODO.md` を更新する
  - `[ ]` → `[x]` に変更
  - `[YYYY-MM-DD HH:MM 完了]` を見出しに追記する
  - TODO.md に記載された内容から変更があった場合は、変更内容を説明に追記する
- ユーザーから新しい要望があれば、`TODO.md` に新しいタスクを追加することも許される

## 初期化: `/todo init`

`TODO.md` が存在しない場合は、`/todo init` で初期化する。

以下のテンプレートで `TODO.md` を作成する：

```markdown
@CLAUDE TODO List

## [ ] タスク名
タスクの説明をここに書く。
```

- 作成後、ユーザーにタスクを追加するよう促す
- すでに `TODO.md` が存在する場合は上書きせず、既存ファイルがある旨を伝える

## ユーザーが使うコマンド例

- `/todo init` : TODO.md を新規作成して初期化する（ファイルが無い場合）
- `/todo` : TODO.md を読んでタスクを実行する
- 「TODO を確認して」: TODO.md の現在の状態を確認・報告する
- 「〇〇を TODO に追加して」: 新しいタスクを TODO.md に追記する
